# Moduł: Command Line

Notatki z nauki obsługi interfejsu wiersza poleceń (CLI) w systemach Windows oraz Linux.

### Windows Command Line (CMD)
* **Podstawy:** Nauka nawigacji po systemie plików NTFS za pomocą komend takich jak `dir` (listowanie), `cd` (zmiana katalogu) oraz `type` (odczyt zawartości pliku).
* **Zarządzanie:** Tworzenie i usuwanie plików/folderów prosto z konsoli.

### Windows PowerShell
* **Czym jest:** Bardziej zaawansowana powłoka Windowsa, oparta na obiektach.
* **Cmdlets:** Poznałem strukturę poleceń typu `Verb-Noun` (np. `Get-Help`, `Get-Service`), która pozwala na automatyzację zarządzania systemem.

### Linux Shells (Bash)
* **Standard:** Nauka pracy w powłoce Bash, która jest standardem w świecie serwerów i narzędzi security.
* **Moc terminala:** Łączenie prostych narzędzi za pomocą potoków (`|`) w celu szybkiego przetwarzania danych.


### Dlaczego to ważne w CyberSec?
CLI (Command Line Interface) pozwala na szybkość i automatyzację. Wiele narzędzi do testów bezpieczeństwa nie posiada okienek (GUI) – działają wyłącznie w konsoli. Zrozumienie różnic między PowerShellem a Bashem pozwala mi sprawniej pracować w różnych środowiskach systemowych.
